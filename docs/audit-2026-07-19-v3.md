**One Important security fix needed — president phone "hide" is client-side only.**

`case when m.hide_president_contact and jsonb_array_length(m.inquiry_contacts) > 0 then null else m.creator_phone end` — this also moves the "president shows if he's the only contact" rule server-side where it belongs.

## Part 2 — Your three complaints: root causes found

### 1. "I didn't see the WhatsApp option"

It exists (post-submit tray + Pending Send rows) but you can never meet it, for two stacked reasons:

- **Auto-SMS wins the race.** On submit, `CollectionForm.tsx:136-139` immediately calls `sendReceiptSms` → `window.location.href = 'sms:…'` (`send.ts:59`). On a phone the OS composer takes the screen *before* the tray with the SMS/WhatsApp pair ever paints. The choice UI only becomes visible after you come back from the composer — and it sits below the submit button.
- **The donation is marked "sent" at attempt time, not send time.** Both send helpers call `markSmsSent()` when the link *fires* (`send.ts:60,71`), and Pending Send filters on `sms_sent_at is null` (`donations.ts:86`). So even if you cancel the SMS composer, the donation is gone from the pending tray — the one persistent place the WhatsApp button lives.

**Fix (recommended):**
1. Don't auto-fire; after submit show the send card (like the design's "Open SMS & Send" moment) with **two buttons: Send by SMS (primary, orange) / Send on WhatsApp (secondary)** and the message preview bubble. One extra tap, full control — and the remembered-channel logic can make the *primary button* be your last-used channel.
2. Only `markSmsSent` when a send button is actually tapped (not on auto attempts), so anything not explicitly sent stays in Pending Send with both buttons.
3. Keep the "sends when you press Send" microcopy so volunteers understand nothing goes out silently.

### 2. "Admin looks designed but clicking any tab goes back to the old UI"

Confirmed — the sidebar is **not a layout**, it's a private component inside `MasterLedger.tsx` (`:396-427`). `router.tsx` is a flat route list with no layout route and no `<Outlet/>`; every other admin screen (`/admin/collections`, `/admin/expenses`, …) renders the old narrow `AppShell`. So the console exists on exactly one page.

**Fix — `AdminLayout` route (the standard React Router pattern):**
```
<Route element={<RequireRole role="admin"><AdminLayout/></RequireRole>}>
  <Route path="/admin"              element={<MasterLedgerContent/>}/>
  <Route path="/admin/collections"  element={<CollectionsContent/>}/>
  <Route path="/admin/expenses"     element={<ExpensesContent/>}/>
  … handovers / cash-in-hand / volunteers / admins / transparency / settings
</Route>
```
`AdminLayout` owns: desktop sidebar (persistent, active-item highlight), mobile header (see #3), sign-out, and `<Outlet/>` for the page body. Admin screens drop their AppShell wrapper (content-only components; the shared screens keep AppShell for the volunteer variant). Result: sidebar persists, page switches happen inside the console, active tab highlights — desktop feels like one app.

### 3. "Mobile version of admin panel + easy collect button"

The design DOES have a mobile admin (captured this pass, added to design-reference-notes.md): no sidebar — a **sticky console header** ("TREASURER CONSOLE / Master Ledger") with a **horizontally scrollable pill-tab row** (Ledger • Collections • Expenses • Volunteers …) pinned while content scrolls, and a **2×2 stat grid** that adds a 4th "Cash w/ volunteers" card.

Current code hides the sidebar on mobile (`hidden lg:flex`) and puts the nav card-grid *below* the whole dashboard — that's why you scroll forever to reach "Collect donation".

**Fix — AdminLayout responsive behavior:**
- **Mobile (<lg):** sticky top: console title + pill-tab row (all sections, active pill orange, `overflow-x-auto`, scroll-snap). Plus a **fixed bottom-right FAB "🪔 Collect"** (or a slim bottom bar: Ledger / Collections / Collect (center, emphasized) / Expenses / More) → `/collect` one tap from anywhere. My recommendation: pill tabs on top + Collect FAB — matches the design language and avoids duplicating nav.
- **Desktop (lg+):** persistent sidebar; put **"🪔 Collect donation" as the first sidebar item styled as a primary button** (above Dashboard), so it never needs scrolling there either.
- Adopt the 4th stat card (Cash w/ volunteers) on mobile; it's already computed by `reconcile.ts`.

### Other UX findings from this pass (fix in the same wave)

1. **Landing "sample report" still has no in-page back** — the one screen F8 called out; add a floating close/back on `/transparency/demo` (and the real public report gets a "shared by <mandal>" home link only when opened from inside the app).
2. **Volunteer tab bar vanishes on Expenses/Handover** — "More" targets `/volunteer/expenses` but those screens don't mount `VolunteerTabBar`; mount it there (or make More a bottom sheet with Expenses/Handover/Language options).
3. Desktop non-dashboard admin pages are narrow centered columns beside empty space — solved automatically by AdminLayout (content width can go to `max-w-4xl` inside the console).
4. Mobile `MobileTopBar` has no menu affordance — replaced by the pill-tab header.
